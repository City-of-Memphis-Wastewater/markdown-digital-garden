```python
class Point:
    point_set = set()

    def __init__(self):
        self.sid = None
        self.zd = None
        self.ip_address = None
        self.idcs = None
        self.rjn_siteid = None
        self.rjn_entityid = None
        self.rjn_name = None
        Point.point_set.add(self)

    def populate_eds_characteristics(self, *, ip_address, idcs, sid, zd):
        self.ip_address = ip_address
        self.idcs = idcs
        self.sid = sid
        self.zd = zd
        return self  # enables chaining

    def populate_rjn_characteristics(self, *, rjn_siteid, rjn_entityid, rjn_name):
        self.rjn_siteid = rjn_siteid
        self.rjn_entityid = rjn_entityid
        self.rjn_name = rjn_name
        return self  # enables chaining

    @classmethod
    def all_points(cls):
        return cls.point_set
```

You got it ‚Äî chained method calls are super clean, expressive, and Pythonic. They let you call multiple methods on the **same object** in a single line, like a fluent sentence. Here's how it works:

---

### üß™ Example 1: Simple Class with Chained Methods

```python
class Builder:
    def __init__(self):
        self.parts = []

    def add_wheel(self):
        self.parts.append("wheel")
        return self

    def add_engine(self):
        self.parts.append("engine")
        return self

    def add_door(self):
        self.parts.append("door")
        return self

    def build(self):
        return f"Building: {', '.join(self.parts)}"
```

### ‚úÖ Chained usage:

```python
car = Builder().add_wheel().add_engine().add_door().build()
print(car)
# Output: Building: wheel, engine, door
```

---

### üîÅ What's going on?

- Each method (`add_wheel()`, `add_engine()`‚Ä¶) **modifies the object**
    
- Each method returns `self`, so the next call continues on the same object
    

---

### üß± Core Terms

|**Term**|**What It Means**|
|---|---|
|**Instantiation**|Creating an instance of a class (e.g. `p = Point()`)|

|**Dunder**|Slang for ‚Äúdouble underscore‚Äù (e.g. `__init__`, `__str__`)|
|**Magic Method**|Another name for dunder methods ‚Äî Python's built-in hooks|
|**Post-init Setup**|Any methods you call after `__init__`, like `populate()` or `configure()`|
|**Chaining**|Calling multiple methods in one expression by returning `self`|
